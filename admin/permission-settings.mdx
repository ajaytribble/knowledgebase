This guide explains how to manage user permissions and access controls in Tribble.

## User Roles

Tribble offers several user roles with different levels of access:

### Admin
- Full system access
- Can manage all settings and configurations
- Can add/remove users
- Can manage all content sources

### Manager
- Can manage team members
- Can manage content sources
- Can view analytics
- Cannot modify system settings

### User
- Basic access to use Tribble
- Can view assigned content
- Can create and manage their own projects
- Cannot modify system settings

## Managing User Access

### Adding Users
1. Navigate to Settings > Users
2. Click "Add User"
3. Enter user email and select role
4. Send invitation

### Modifying User Roles
1. Go to Settings > Users
2. Find the user
3. Click "Edit"
4. Select new role
5. Save changes

### Removing Users
1. Go to Settings > Users
2. Find the user
3. Click "Remove"
4. Confirm removal

<Warning>
Removing a user will revoke their access immediately. Make sure to transfer any important data or projects before removal.
</Warning>

## Content Access Control

### Private vs Public Content
- Public: Accessible to all users
- Private: Restricted to specific users or groups

### Managing Content Access
1. Navigate to Sources
2. Select the content
3. Click "Manage Access"
4. Choose visibility (Public/Private)
5. If private, select users or groups

## Best Practices

- Regularly review user access levels
- Remove inactive users promptly
- Use groups for easier permission management
- Document permission changes
- Maintain principle of least privilege
