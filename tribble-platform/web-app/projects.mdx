Navigate to the Projects tab in the [web app](https://my.tribble.ai) to begin creating your project.

## Select Your Document Type

<Tabs>
  <Tab title="Spreadsheet Documents">
    ### Project Creation
    
    <Frame caption="Projects Dashboard">
      <img src="/images/web-app/creating%20projects/projects-dashboard.png" alt="Projects Dashboard View" />
    </Frame>

    Upload your questionnaire in XLSX (Excel spreadsheet) format.

    <Steps>
      <Step title="Start Project Creation">
        Click the "Create Project" button and select "Upload Questionnaire"
      </Step>
      <Step title="Upload Document">
        Upload your Excel spreadsheet
      </Step>
      <Step title="Enter Project Details">
        Fill in the required information:
        - Project title
        - Customer name
        - Due date
        - Project owner
        
        Optional:
        - Link to Salesforce opportunity
      </Step>
      <Step title="Configure Settings">
        1. Select answer length
        
        2. Optionally apply tags to focus responses on specific:
           - Product lines
           - Geographic regions
           - Industry verticals
           - Compliance frameworks
        
        <Info>
          [Click here](/admin/tags) if you're an admin and want to learn more about tags
        </Info>
      </Step>
    </Steps>

    ### Document Processing
    <Steps>
      <Step title="Initial Analysis">
        Tribble analyzes your spreadsheet to identify question and answer columns/cells
      </Step>
      <Step title="Review & Verify">
        Review Tribble's analysis:
        - Verify identified question columns and cells
        - Confirm answer column locations
        - Add or remove question/answer ranges as needed
      </Step>
      <Step title="Confirm & Generate">
        Once you're satisfied with the document setup:
        - Click Continue to proceed
        - Tribble will generate responses for all questions using your knowledge base
      </Step>
    </Steps>

    <Frame caption="Spreadsheet Review Interface">
      <img src="/images/web-app/creating%20projects/spreadsheet-review.png" alt="Spreadsheet Column Review and Verification" />
    </Frame>

    ### Workflows

    After Tribble generates initial answers, click the review cell next to any satisfactory answer to mark it as reviewed. For answers that need modification:

    #### Editing Answers
    Two ways to edit answers:

    1. **Direct Edit**
       - Click the pencil icon on any question/answer pair
       - Make your changes directly in the editor
       - Changes are automatically marked as reviewed

    2. **Chat with Tribble**
       - Choose from suggested actions or chat with Tribble
       - Request longer answers, tone changes, or other modifications
       - Replace the answer when satisfied
       - Modified answers are automatically marked as reviewed

    <Frame caption="Chat with Tribble">
      <img src="/images/web-app/workflows/chat-interaction.gif" alt="Demonstrating chat and suggested actions with Tribble" />
    </Frame>

    #### Collaboration

    <CardGroup cols={3}>

  <Tab title="Long-form Documents">
    ### Project Creation
    
    <Frame caption="Projects Dashboard">
      <img src="/images/web-app/creating%20projects/projects-dashboard.png" alt="Projects Dashboard View" />
    </Frame>

    Upload your RFx documents in DOCX (Microsoft Word) or PDF format. You can upload one or multiple documents for your proposal.

    <Steps>
      <Step title="Start Project Creation">
        Click the "Create Project" button and select "Upload Questionnaire"
      </Step>
      <Step title="Upload Documents">
        Upload one or more documents in DOCX or PDF format
      </Step>
      <Step title="Enter Project Details">
        Fill in the required information:
        - Project title
        - Customer name
        - Due date
        - Project owner
        
        Optional:
        - Link to Salesforce opportunity
      </Step>
      <Step title="Configure Settings">
        Optionally apply tags to focus responses on specific:
        - Product lines
        - Geographic regions
        - Industry verticals
        - Compliance frameworks
        
        <Info>
          [Click here](/admin/tags) if you're an admin and want to learn more about tags
        </Info>
      </Step>
    </Steps>

    ### Document Processing
    <Steps>
      <Step title="Document Analysis">
        Tribble analyzes your RFx documents and their content
      </Step>
      <Step title="Review & Customize">
        - Review the generated proposal outline
        - Modify section headers and descriptions
        - Add or remove sections as needed
        - Ask questions about the RFx content
        - Review any triggered go/no-go criteria warnings
        
        <Info>
          [Learn more about configuring go/no-go criteria](/admin/go-no-go)
        </Info>
      </Step>
      <Step title="Finalize Structure">
        Once you're satisfied with the proposal structure:
        - Click Continue to proceed
        - Tribble will generate content for each section
      </Step>
    </Steps>

    <Frame caption="Proposal Structure Review">
      <img src="/images/web-app/creating%20projects/proposal-review.png" alt="Proposal Structure Review and Customization" />
    </Frame>

    ### Workflows

    #### Editing Answers
    <Steps>
      <Step title="Section Navigation">
        - Navigate through proposal sections
        - Use the outline view for quick access
        - Edit section content directly
      </Step>
      <Step title="Content Editing">
        - Use rich text editor for formatting
        - Insert images and tables
        - Apply document styles
      </Step>
      <Step title="Version Control">
        - Track changes by section
        - View edit history
        - Restore previous versions
      </Step>
    </Steps>

    #### Verifying Answers
    <Steps>
      <Step title="Section Review">
        - Mark sections for review
        - Track review status
        - Add review comments
      </Step>
      <Step title="Compliance Check">
        - Verify requirements coverage
        - Check against RFx criteria
        - Ensure consistent messaging
      </Step>
    </Steps>

    #### Collaboration
    <Steps>
      <Step title="Team Coordination">
        - Assign section owners
        - Set review deadlines
        - Track section progress
      </Step>
      <Step title="Review Process">
        - Conduct collaborative reviews
        - Manage approval workflows
        - Consolidate feedback
      </Step>
    </Steps>
  </Tab>
