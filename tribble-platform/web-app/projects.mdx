## Creating a New Project

<Frame caption="Create New Project">
  <img src="/images/web-app/creating%20projects/new-project-form.png" alt="New Project Form" />
</Frame>

### Option 1: Upload a Document

<Steps>
  <Step title="Start New Project">
    Click the "Create Project" button and select "Upload Questionnaire"
  </Step>
  <Step title="Choose File">
    Upload your document in DOCX, PDF, or XLSX format
  </Step>
  <Step title="Enter Project Details">
    - Project title
    - Customer name
    - Due date
    - Project owner
  </Step>
  <Step title="Configure Settings">
    - Select relevant tags (product, region, etc.)
    - Set answer length preferences
    - Enable/disable "Terse yes or no answers" option
  </Step>
</Steps>

<Frame caption="Project Creation Process">
  <img src="/images/web-app/creating%20projects/project-filled.gif" alt="Project Creation Process" />
</Frame>

### Option 2: Manual Question Entry

<Steps>
  <Step title="Create Project">
    Click "Create Project" and select "Manual Entry"
  </Step>
  <Step title="Set Up Project">
    Enter project details and configure settings
  </Step>
  <Step title="Add Questions">
    Use "Add Questions" button to input or copy-paste questions
  </Step>
  <Step title="Organize Content">
    Group questions by function (Legal, Security, Product, etc.)
  </Step>
</Steps>

## Project Settings

### Answer Length Options
<AccordionGroup>
  <Accordion title="Default (~100 words)">
    Standard response length suitable for most questions
  </Accordion>
  <Accordion title="Shorter (~50 words)">
    Concise responses for simple questions or requirements
  </Accordion>
  <Accordion title="Longer (~150 words)">
    Detailed responses for complex questions or technical requirements
  </Accordion>
</AccordionGroup>

### Project Tags
<Note>
Tags help confine Tribble's responses to relevant content. Use them to specify:
- Product lines
- Geographic regions
- Industry verticals
- Compliance frameworks
</Note>

## Best Practices

### Project Setup
<Tip>
- Use clear, descriptive project names
- Set realistic deadlines
- Assign appropriate owners
- Apply relevant tags
</Tip>

### Document Preparation
<Warning>
Before uploading documents:
- Clean up formatting
- Remove unnecessary content
- Verify question structure
- Check for completeness
</Warning>

## Project Organization

### Grouping Questions
- Organize by function
- Create logical sections
- Use consistent naming
- Apply relevant tags

### Team Collaboration
- Assign section owners
- Set review workflows
- Track progress
- Maintain version control
